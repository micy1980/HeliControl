MGM Reporting - riport előtti fejlesztési checklist

Állapotfelmérés és feladatlista a Management és Konszolidációs riport elkészítése előtt.

P0 Riportot blokkoló hibák és adatbiztonsági kockázatok
P1 Riport előtt javasolt stabilizálás
P2 Kényelmi, egységesítési és UX javítások
P3 Riport után is ráérő, de nyilvántartott tételek

1. Riport Előtti Blokkolók

Ezeket a riport felület építése előtt érdemes lezárni, mert közvetlenül érintik az adatok megbízhatóságát vagy a riport alapját.

Kész Prioritás Terület Feladat
P0 GL import validáció A Tartozik/Követel és Balance eltérés minden esetben hard fájlhiba legyen. Az import ne kerülhessen be, ha az egyenleg nem egyezik két tizedes pontossággal.
P0 GL import validáció A TFORG, KFORG, TEgy, KEgy, Egyenleg és minden importált számmező egységesen két tizedessel legyen értelmezve, tárolva és megjelenítve.
P0 COA import validáció Duplikált GL szám az import fájlban hard hiba legyen, és még az adatbázis-felülírás előtt meg kell fogni. Nem maradhat olyan eset, ahol csak az egyik sor kerül be.
P0 Főkönyvi kivonat A főkönyvi kivonat validációs végpont és UI működjön stabilan. Ne jelenjen meg "Nincs ilyen végpont" üzenet.
P0 Főkönyvi kivonat A főkönyvi validáció csak aktív főkönyvi adatokból dolgozzon, és a Management/Konszi riportkód hiányát csak akkor jelezze, ha az adott GL-en bármelyik hónapban van érték.
P0 Import session Minden import kapjon egyértelmű import azonosítót/batch id-t, és ez jelenjen meg COA, GL, részletek és előzmények nézetben is.
P0 Import előzmények A COA import és GL import log/előzmény ne keveredjen. A számlatükörben COA importok, a GL importnál GL importok legyenek, vagy legyen külön szűrt import log menü.
P0 Report törzsadat A Management és Konszolidációs riport törzsek legyenek végleges, egységes táblázatos szerkesztőlogikán. Mentés, import template és csoporttörzs választás hibamentesen működjön.

2. Ablakkezelés És Layout Stabilizálás

Kész Prioritás Terület Feladat
P1 Belső ablakok A teljes méret / visszaállítás gomb ténylegesen töltse ki a munkaterületet, ne csak vizuálisan változzon.
P1 Belső ablakok Dupla kattintás a belső ablak fejlécén váltson teljes méret és visszaállított méret között.
P1 Belső ablakok Átméretezés működjön mind a négy oldalon és a sarkokon is, ne csak jobb alsó sarokban.
P1 Belső ablakok A kezdő ablakméretek legyenek nagyobbak a sokoszlopos felületeken: COA import, GL import, import részletek, validációs lista, sorszabályok.
P1 Belső ablakok Státuszváltozás, validáció, figyelmeztetés vagy hiba ne változtassa meg az ablak méretét és pozícióját.
P1 Renderelés A programon belüli villogást, teljes újrarenderelési ugrásokat meg kell szüntetni. Mentés, pipa, validáció és törlés ne villantsa újra az egész ablakot.
P2 ESC működés ESC először nyitott oszlopválasztót vagy részpanelt zárjon, csak utána a legfelső belső ablakot.

3. Scroll, Táblázat És Oszlopkezelés

Kész Prioritás Terület Feladat
P1 Táblázatok Minden sokoszlopos táblázatban működjön a függőleges és vízszintes scroll. A COA import preview és GL import preview lefelé is görgethető legyen.
P1 Táblázatok A fejléc maradjon látható, de ne akadályozza a táblatest scrollját. A táblázatkonténereknél min-height: 0 / overflow logikát egységesíteni kell.
P1 Táblázatok Felesleges vízszintes scroll ne jelenjen meg sorszabályoknál és egyszerű listáknál. Scroll csak akkor legyen, ha tényleg nem fér ki az adat.
P1 Oszlopkezelés Minden oszlopkezelős táblában lehessen kézzel oszlopot átrendezni. Kivétel: az 1-12 havi oszlopok sorrendje kötött.
P1 Oszlopkezelés Oszlopsorrend, oszlopszélesség és láthatóság user settingként automatikusan mentődjön.
P2 Oszlopkezelés Az Oszlopok menüben legyen "Nézet alaphelyzetbe állítása" funkció.
P2 Oszlopkezelés Ha minden opcionális oszlop ki van kapcsolva, a tábla ne csússzon szét és ne jelenjen meg üres, aránytalan felület.
P2 Táblázatok Sormagasság egységesítése minden táblában. A csoporttörzs kompakt sormagassága legyen az alap.
P2 Űrlap kontrollok Checkbox méretek egységesítése minden modulban, beleértve a validációs szabályok ablakot is.

4. Számlatükör Import

Kész Prioritás Terület Feladat
P1 Validáció Sikeres validáció után ne nyíljon automatikus eredményablak. A zöld státuszbar jelezze: "Validálva, importálható".
P1 Validáció Hibánál/piros állapotnál magából a státuszból legyen egyértelmű, hogy az import nem indítható. Részletek csak gombnyomásra nyíljon.
P1 Validációs lista A figyelmeztetés/hiba lista kapjon GL név oszlopot, hogy azonos GL név figyelmeztetésnél látszódjon a számla neve is.
P1 Preview A preview-ban a megfeleltetett oszlopok azonnal kapjanak színezést, amikor a mapping változik, nem csak validáció után.
P1 Preview A preview-ban minden importfájl oszlop látható legyen és jobbra is ki lehessen scrollozni.
P1 Sorszabály Sorszabály pipa változásakor azonnal frissüljenek a darabszámok: importálható, összesítő, kimaradó, inaktív.
P2 Sorszabály A "Kimaradó összesítő" felirat egyszerűsíthető vagy elhagyható, mert a kivétel logikája a pipa alapján egyértelmű.
P1 Import gomb Az "Import indítása" gomb a validációs gomb mellé kerüljön, és csak validált/importálható állapotban legyen érdemben használható.
P2 Template COA import template mentés, kiválasztás és alkalmazás ne mozgassa el az ablakot, és company-specifikus működés szerint tárolódjon.

5. GL Import És Import Részletek

Kész Prioritás Terület Feladat
P0 Számformátum Minden GL import számmező két tizedessel jelenjen meg: mapping preview, validációs lista, import részletek és főkönyvi kivonat kapcsolat.
P0 Fájlhiba A Tartozik/Követel és Balance eltérés fájlhibaként állítsa meg az importot. A hiba a validációs listában legyen részletezve.
P1 Validáció A GL import mapping felületen ugyanolyan státuszbar legyen, mint a COA importnál: zöld/sárga/piros állapot, Részletek gombbal.
P1 Validáció A validációs popup ne listázza hosszan a hibákat automatikusan. Legyen rövid státusz, a részletek külön validációs listában.
P1 Soft hiba Az import részletekben legyen "Minden soft hiba OK" gomb, amely a kezelhető figyelmeztetéseket egyszerre elfogadja.
P1 Gyors művelet A validációs listában később legyen gyors művelet: hiányzó GL felvétele, számlatükör név frissítése, soft hiba OK.
P1 Életciklus Az import részletek tetején egyértelműen látszódjon: importálva, validálva, aktiválható, aktív főkönyvben, GL-ből törölve, felülírt.
P1 Újraaktiválás Ha a GL adatok törölve lettek, az import session maradjon meg "GL-ből törölve" állapotban, és újraaktiválható legyen validáció után.
P2 Lista GL import lista kapjon szűrőket: év, hónap, státusz, főkönyvben igen/nem, fájlnév, hibás/figyelmeztetéses.

6. Számlatükör És Tömeges Módosítás

Kész Prioritás Terület Feladat
P1 Validáció Hiányzó Management riportkód, Konszi riportkód és BS/PL csak akkor legyen hiba, ha az adott GL-en van bármelyik hónapban érték.
P1 Tömeges módosítás A tömeges módosítás ablakból a külön "mező módosítása" pipák kikerülnek. Csak kitöltött mezők módosítanak.
P1 Tömeges módosítás Ha a felhasználó módosítást indít, de nincs tényleges változás, ne történjen adatírás. Üzenet: "Nem történt módosítás".
P1 Audit log Egyedi és tömeges törzsadat módosításokról részletes audit log kell: ki, mikor, melyik cég, melyik tábla, melyik mező, miről mire változott.
P2 Keresés Wildcard keresés működjön: 53, 53*, *53, *53*. Ez minden szűrős listában egységes legyen.

7. Report Törzsadatok

Kész Prioritás Terület Feladat
P1 Management törzs Report kód tábla és Csoport1/2/3 törzsek egységes kompakt táblázatos szerkesztőlogikával működjenek.
P1 Konszi törzs A Konszolidációs riport törzs ugyanazt a kinézetet és működést kapja, mint a Management riport törzs.
P1 Mentés Kód és megnevezés duplikációjánál látható hibaüzenet jelenjen meg. Ne csak villanjon vagy csendben ne mentsen.
P1 Import template Management riport törzs, Konszi riport törzs és csoporttörzsek kapjanak letölthető Excel/CSV template-et. A csoport template-ben legyen csoport oszlop.
P2 BS/PL Át kell gondolni, hogy BS/PL maradjon-e külön Management és Konszi törzsben is, vagy a számlatükör/riportkód legyen az egyetlen forrás.
P2 Company scope A report törzsadatok company-specifikusak legyenek. Ne globális törzsadatként működjenek.

8. Főkönyvi Kivonat

Kész Prioritás Terület Feladat
P1 Adatforrás Csak aktív GL import sessionből származó főkönyvi adat jelenjen meg.
P1 Havi import info Hónap fejlécére kattintva vagy hoverre jelenjen meg az aktív import azonosítója, fájlnév, importáló és aktiválás dátuma.
P1 Számformátum Főkönyvi kivonat számai egységesen kezeljék a globális egységet, de az alapadat továbbra is 1-es egységben maradjon.
P1 Scroll 12 havi nézetben vízszintes és függőleges scroll, fix havi sorrend, stabil fejléc.
P2 Export CSV export után később Excel export is kell formázással.

9. Admin, Naplózás És Adatkarbantartás

Kész Prioritás Terület Feladat
P1 Naplók Naplók legyenek típusonként elkülönítve: rendszer, import, validáció, törzsadat/audit, admin, biztonság.
P1 Naplók SA és beégetett lenkei.peter láthassa az összes cég összes naplóját, de normál nézetben naplók company-specifikusak legyenek.
P1 Adatkarbantartás Törlés után az áttekintés/dashboard automatikusan frissüljön, ne kelljen elkattintani és vissza.
P1 Adatkarbantartás Az aktív cég / aktív év törlése egyértelműen scope-olt legyen. Company törlés külön a Cég menüben, teljes company id alatti adatkörrel.
P2 Jogosultság Globális lenkei.peter tartalék user sehol ne jelenjen meg, de SA szintű bejelentkezésként működjön.

10. Admin És Rendszerbiztonság - Riport Előtti Blokkoló Kör

Döntés: riportfejlesztés csak akkor induljon, ha az alap admin, session, jogosultság és biztonsági beállítások stabilan működnek. A következő kör fókusza: jogosultsági mátrix, naplózás keményítése, SMTP/email, session-security finomhangolás és 2FA-kompatibilis struktúra.

Frissítve: 2026-05-21. Jogosultsági mátrix első köre elkészült külön Admin ablakban, új jogosultsági táblákkal és backend guarddal az admin, beállítás, backup, napló, licensz, validáció, riport/trial balance, GL import, COA, budget, árfolyam és riport törzsadat fő útvonalain. A mátrix UI-ban a nem értelmezett műveletek checkbox helyett kötőjelet kapnak; SA-ként a magasabb alap szerepkörű jogok is adhatók, checkbox mellett ADMIN/SA minimum jelöléssel. Az explicit SA által mentett mátrixjog backend oldalon is felülírja a user alap szerepkörét. A frontend menü, modulindító és fő műveleti gombok a bootstrapben kapott effektív jogosultságokból dolgoznak. A naplózás keményítése első köre is elkészült: user, cég, beállítás, címke, validációs szabály, summary rule és jogosultság módosítások olvasható régi érték -> új érték auditot kapnak; session revoke és jelszó reset session zárás részletezettebb auditként kerül be. Az SMTP beállításokhoz teszt email küldés, validáció és audit naplózás készült, a jelszó auditban maszkolva marad, a mentett SMTP jelszó és a licensz secret pedig nem kerül vissza a böngészőbe. Az email értesítések első köre is elkészült: címzettlista, eseménykapcsolók és teszt értesítés az Email beállítások alatt; bekötve az automatikus backup hiba, lockolt user, restore és jogosultság/security esemény, a kritikus import hiba kapcsoló előkészítve.

Aktuális prioritási sorrend riport előtt

  1. P0 - Session/security beállítások: a beállításokban megadott timeout, session lejárat, hibás login limit, zárolási idő és automatikus feloldás tényleges backend összekötése az aktív session listával.
  2. P0 - Fiókok és zárolási UX: zárolt fiókok, login kísérletek, kézi feloldás, státuszok és user session kiléptetés ellenőrzése/rendezése.
  3. P1 - Napló szűrés lezáró audit: wildcard, ékezetfüggetlen keresés, minden oszlop, stabil fókusz és látható/szűrt naplók törlése; a checklist ellentmondó státuszainak tisztítása.
  4. P0 - Címkerendszer audit: hardcode-olt feliratok átnézése, a katalógusban már szereplő címkék tényleges bekötése.
  5. P0 - Nyelvválasztó: magyar/angol váltás tényleges használata, mentése és UTF-8/ékezethelyesség ellenőrzése.
  6. P1 - Téma: nappali/éjszakai mód úgy, hogy a táblázatok, checkboxok, státusz badge-ek, ablakfejlécek és alsó státuszsáv ne törjenek.
  7. P2 - 2FA előkészítés: TOTP-kompatibilis auth/session struktúra, admin reset helye, recovery kódok és audit előkészítése teljes bekényszerítés nélkül.
  8. P0 - Riport előfeltételek: valid COA, aktív GL import, riportkód-hiány validáció csak értékes GL-ekre, Management/Konszi riport törzs feltöltés-validálás-COA hozzárendelés.
  9. P1 - Riportépítés: Management riport, Konszi riport, subtotal/BS-PL/havi/YTD/éves összesen logika és Excel export.
Kész Prioritás Terület Feladat
P0 Session Automatikus kijelentkezés inaktivitás alapján, abszolút session lejárat, last seen frissítés, lejárt vagy visszavont session tiltása.
P0 Aktív sessionök Admin ablak az aktív munkamenetekhez: user, role, cég, belépés, utolsó aktivitás, lejárat, IP/user agent, kiléptetés userenként vagy sessionönként.
P0 Login védelem Sikertelen belépési limit, fiók zárolása, automatikus feloldás beállítható idő után, kézi admin feloldás, minden esemény security naplóban.
P0 Felhasználók User módosítás, aktiválás/inaktiválás, zárolt állapot kijelzése, feloldás, jelszó reset, kötelező jelszócsere, user sessionök kiléptetése.
P0 Cégek Cég szerkesztése, aktív/inaktív állapot, alapdeviza és megjelenítési adatok előkészítése, logo mező/feltöltés előkészítése, company törlés dupla védelemmel.
P0 Számformátum Rendszer szintű számformátum: ezres elválasztó, tizedesjel, GL tizedes, riport tizedes, árfolyam tizedes, negatív szám formátum.
P0 Idő és időzóna Programidő beállítása gép órája vagy timeserver alapján, Windows-szerű időzóna választó, automatikus nyári időszámítás kapcsoló, timeserver szinkron állapot, sikeres/sikertelen elérés visszajelzése és backup időbélyeg bekötése.
P0 Jogosultság Jogosultsági mátrix külön Admin ablakban elkészült. Modul + művelet szinten kezeli: megtekintés, import, validálás, aktiválás, módosítás, törlés, export, admin és restore jogosultság.
P0 Cégszintű jogosultság User-cég hozzárendelés első köre elkészült: láthatja/kezelheti jogok, cégváltó és fő admin/riport/GL/COA/budget útvonalak figyelembe veszik. A további riport-specifikus részjogok a riportépítéskor finomíthatók.
P0 Jogosultság backend Backend guard első és második köre elkészült az admin, beállítás, backup, napló, licensz, validáció, riport/trial balance, GL import, COA, budget, árfolyam, import template, sorszabály és riport törzsadat fő műveleteire. SA nem szűkíthető mátrixszal, saját mátrix mentése tiltott, a globális lenkei.peter tartalék SA nem jelenik meg normál userként.
P1 SMTP Email/SMTP beállítás külön admin ablakban: host, port, TLS/SSL, user, jelszó, feladó és teszt email elkészült. A teszt eredménye egyértelmű visszajelzést ad, siker és hiba esetén is admin/audit naplóba kerül; az SMTP jelszó az audit részletben maszkolt, a kliens felé nem kerül visszaküldésre, üres mezővel a mentett jelszó marad. A settings API a licensz secretet is maszkolja.
P1 Email értesítések Email értesítések első köre elkészült: címzettlista, eseménykapcsolók és teszt értesítés az Email beállítások alatt. Bekötve: automatikus backup hiba, lockolt user, restore és jogosultság/security esemény; kritikus import hiba kapcsoló előkészítve.
P0 Naplózás Security/admin audit keményítése első köre elkészült: user, cég, törzsadat, címke, validációs szabály, summary rule, beállítás és jogosultság változásoknál régi érték -> új érték mentés van. Login hiba, lock/unlock, session revoke, restore és naplótörlés külön audit/security eseményként szerepel.
P1 Napló szűrés Napló szűrés tovább finomítása: wildcard, ékezetfüggetlen keresés, minden oszlop, stabil fókusz, szűrt törlésnél pontosan a látható/szűrt halmaz kezelése.
P0 Session beállítások Automatikus kijelentkezési idő, inaktív session kezelés, invalid login limit, zárolási idő és automatikus feloldás legyen összekötve a Beállításokban megadott értékekkel és az aktív session listával.
P1 Backup Automatikus backup egyszeri/napi/heti/havi ütemezéssel, rendezett backup felület, állítható és tallózható mentési útvonal, retention beállítás, backup fájl letöltés, útvonal másolás, SA restore jogosultság, tallózott restore fájlból, restore előtti biztonsági mentés, védett törlés és cégtörlés előtti opcionális automatikus mentés.
P2 2FA TOTP alapú 2FA előkészítése úgy, hogy később ne kelljen szétszedni az auth/session struktúrát: SA/Admin kötelező mód, recovery kód, 2FA reset adminból, tartalék SA külön kezelése és naplózása.

11. Beállítások, Nyelv És Felület Egységesítés

Döntés: a riport fejlesztése előtt az admin és beállítási felületeknek egységes szerkezetben, stabil táblázatokkal, magyar/angol címkerendszerrel és használható témaváltással kell működniük. A Beállítások ablakban csak valódi rendszerbeállítási fülek maradnak; a Cégek, Felhasználók és Munkamenetek külön admin ablakok.

Frissítve: 2026-05-20. Lezárt commitok: 7164e21 admin táblák/címkék rendezése, f10faf8 napló szűrés és validációs szabályok UX javítása.

Kész Prioritás Terület Feladat
P0 Beállítások struktúra A Beállítások ablakból kikerültek a felesleges fő fülsávok. A Beállítások alatt Rendszer, Email beállításai, Törzsadatok, Címkék és nyelv maradt; a Cégek, Felhasználók és Munkamenetek külön admin ablakban nyílnak.
P0 Beállítások UX A felesleges felső fülsávok kerüljenek ki a beállítások, vállalatok és felhasználók felületeiről ott, ahol már külön belső navigáció vagy ablakcím van. A kezdő ablakméret minden admin ablaknál akkora legyen, hogy az alap tartalom ne essen szét.
P0 Vállalatok A Vállalatok ablak normalizált méretet, rendezett gombelhelyezést és állítható oszlopszélességet kapott. A logo feltöltés megmaradt; külön logo törlés vagy további cégmódosítási naplózás igény esetén külön feladatként kezelendő.
P0 Fiókok Felhasználók és aktív sessionök egységes táblázatos felületet kaptak állítható oszlopokkal. Zárolt fiókok és login kísérletek külön áttekintése még ellenőrizendő/rendezendő.
P0 Naplók Naplókban minden adat-oszlopra van fejléces szűrés, wildcard támogatással és ékezetfüggetlenebb kereséssel. A látható naplók törlése megerősítéssel fut, és maga a törlés is audit/security naplóba kerül.
P0 Validációs szabályok A validációs szabályok ablakban nagyobb és középre igazított checkbox/select kontrollok vannak. A technikai kulcsok nem jelennek meg, és a két tábla Szint oszlopa vizuálisan egységesebb lett.
P0 Címkerendszer Elkészült a címke admin felület modul/menü bontással, zárolt kulccsal, szerkeszthető magyar/angol címkével és export/import funkcióval. Az alap címkekatalógus 182 elemre bővült.
P0 Címkerendszer audit A teljes alkalmazásban még át kell nézni a hardcode-olt feliratokat, és a címkerendszerrel ténylegesen be kell kötni azokat a szövegeket is, amelyek most még csak katalógusban szerepelnek.
P0 Nyelv Legyen nyelvválasztó legalább magyar és angol nyelvvel. A kiválasztott nyelv user vagy rendszer beállításként tárolódjon, és minden új UI címke UTF-8 kompatibilisen, ékezethelyesen jelenjen meg.
P1 Téma Nappali és éjszakai mód legyen választható. A témaváltás ne törje a táblázatokat, checkboxokat, státusz badge-eket, ablakfejléceket és alsó státuszsávot.
P1 Táblázatok Központi oszlopszélesség-állítás került a táblázatokhoz, a régebbi kézzel épített admin/menü táblákra is automatikusan ráül. A checkbox/státusz igazítást és sormagasságot a validációs és admin táblákon külön is finomítottuk.

12. Riport Építés Előfeltételei

Javasolt sorrend: előbb admin/rendszerbiztonság, adatbiztonság és validáció, utána törzsadat és főkönyvi kivonat stabilizálása, csak ezután jöjjön a tényleges Management/Konszi riport felület.
Kész Prioritás Terület Feladat
P0 Adatalap Legyen aktív, valid COA a cégben, duplikált GL szám nélkül, rendezett import sorrenddel.
P0 Adatalap Legyen aktív GL import legalább egy teszthónapra, fájlhiba nélkül, egyértelmű import azonosítóval.
P0 Adatalap A főkönyvi kivonat validációja tudja jelezni a riportkód hiányokat, de csak értékes GL-ekre.

13. Későbbi, Nem Riport-Blokkoló Kör

Adatkapcsolatok

  • MNB árfolyam letöltés bekötése.
  • Árfolyam import/validáció finomítása.
  • Budget / Forecast import módszertan GL import mintára.

Üzemeltetés

  • Backup / restore későbbi finomítás: opcionális mappa megnyitás, titkosított backup és email riasztás.
  • Évváltás folyamata.
  • Windows szolgáltatás vagy egykattintásos indító.
  • Jogosultsági mátrix riport utáni finomhangolása, ha a riport jogosultságokhoz további részletszabály kell.
